# Component Examples

This page demonstrates all the available MDX components for the Kaira Chat SDK documentation.

## Syntax Highlighting

All code blocks are automatically highlighted using **Shiki** at build time. Supports both light and dark themes.

### TypeScript

```typescript
import { ChatEngine } from '@kaira/chat-core';
import { PollingTransport } from '@kaira/chat-transport-polling';

const engine = new ChatEngine({
  transport: new PollingTransport({
    intervalMs: 1000,
    poll: async () => [],
    send: async () => {},
  }),
});

engine.on('message:received', (event) => {
  console.log('New message:', event.message);
});

await engine.sendMessage('demo-conversation', {
  content: 'Hello, world!',
  type: 'text',
});
```

### JavaScript

```javascript
import { ChatEngine } from '@kaira/chat-core';
import { PollingTransport } from '@kaira/chat-transport-polling';

const engine = new ChatEngine({
  transport: new PollingTransport({
    intervalMs: 1000,
    poll: async () => [],
  }),
});

engine.on('connection:state', (event) => {
  console.log('Connection:', event.state);
});

async function sendHello() {
  await engine.connect();
  await engine.sendMessage('demo-conversation', {
    type: 'text',
    content: 'Hello, world!',
  });
}
```

## Tables

Tables are styled automatically:

| Property    | Type                        | Required | Description                          |
| ----------- | --------------------------- | -------- | ------------------------------------ |
| `id`        | `string`                    | Yes      | Unique identifier for the message    |
| `type`      | `MessageType`               | Yes      | Message discriminator                |
| `sender`    | `Participant`               | Yes      | Message sender                       |
| `timestamp` | `number`                    | Yes      | Unix timestamp in milliseconds       |
| `status`    | `MessageStatus`             | Yes      | Delivery lifecycle status            |
| `metadata`  | `MessageMetadata`           | No       | Optional metadata (e.g. clientNonce) |
