# React Integration

`@kaira/chat-react` wraps `ChatEngine` with a provider + focused hooks.

## Provider

```ts
export interface ChatProviderProps {
  readonly engine?: ChatEngine;
  readonly createEngine?: () => ChatEngine;
  readonly children: ReactNode;
  readonly autoConnect?: boolean;
  readonly onConnectError?: (error: Error) => void;
}
```

- Pass `engine` when you already created one.
- Pass `createEngine` for lazy initialization.
- `ChatProvider` requires one of `engine` or `createEngine`; otherwise it throws.
- Set `autoConnect` for mount/unmount lifecycle handling.

## Hooks

- `useMessages(conversationId)` -> live + initial message list
- `useSendMessage()` -> stable send callback
- `useStreamingMessage(conversationId)` -> transient streaming state
- `useTypingState(conversationId)` -> full conversation typing snapshot
- `useTypingParticipants(conversationId, options?)` -> filtered typing participants
- `useIsTyping(conversationId, participantId?)` -> boolean selector
- `useTypingController(conversationId)` -> notify/stop helpers plus support flag
- `useConnectionState()` -> current connection state
- `useConversation(conversationId)` -> conversation lookup
- `useOptimisticMessages(messages)` -> optimistic reconciliation helpers
- `useChatEngine()` -> direct engine access from context

`useStreamingMessage` only clears its local preview when the matching
conversation and active stream message finish or error.

## Example Chat UI

```tsx
'use client';

import type { JSX } from 'react';

import {
  useMessages,
  useOptimisticMessages,
  useSendMessage,
  useStreamingMessage,
  useTypingController,
  useTypingParticipants,
} from '@kaira/chat-react';
import { MessageInput, ThinkingIndicator, TypingIndicator } from '@kaira/chat-ui';

export function ChatPanel(props: { readonly conversationId: string }): JSX.Element {
  const { conversationId } = props;
  const messages = useMessages(conversationId);
  const { mergedMessages } = useOptimisticMessages(messages);
  const sendMessage = useSendMessage();
  const streaming = useStreamingMessage(conversationId);
  const typingParticipants = useTypingParticipants(conversationId);
  const { notifyTyping, stopTyping, isSupported } = useTypingController(conversationId);

  return (
    <section>
      <div>
        {mergedMessages.map((message) => (
          <p key={message.id}>
            <strong>{message.sender.role}:</strong>{' '}
            {message.type === 'text' || message.type === 'ai' ? message.content : message.type}
          </p>
        ))}
      </div>

      {streaming.isStreaming ? <ThinkingIndicator /> : null}
      <TypingIndicator participants={typingParticipants} />

      <MessageInput
        onValueChange={(text) => {
          if (!isSupported) return;
          if (text.trim()) {
            notifyTyping();
            return;
          }

          stopTyping();
        }}
        onBlur={stopTyping}
        onSend={async (text) => {
          stopTyping();
          await sendMessage(conversationId, { type: 'text', content: text });
        }}
      />
    </section>
  );
}
```

`useTypingController().isSupported` reports transport typing capability. The hooks still
work against local engine state when the transport does not support typing updates.
